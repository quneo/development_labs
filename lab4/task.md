Лабораторная работа №4
Цель: создать интерактивное веб-приложение для анализа табличных данных о пассажиропотоке с несколькими взаимозависимыми графиками и элементами управления.

Описание задания:

Использовать набор данных с информацией о пассажиропотоке по станциям метро или автобусным маршрутам (дата, станция, количество пассажиров, время суток).
Создать дашборд с такими элементами:
Выпадающий список для выбора станции.
Слайдер (dcc.Slider) для выбора времени суток (например, часы от 0 до 23).
График временного ряда с динамикой пассажиропотока по выбранной станции и времени.
Тепловая карта (heatmap) с интенсивностью пассажиропотока по станциям и времени суток.
Диаграмма рассеяния (scatter plot) для анализа зависимости пассажиропотока от времени суток.
Обеспечить связь между элементами: изменение станции или времени суток обновляет все графики.
Добавить возможность выбора нескольких станций для сравнения.
Использовать callbacks для обновления данных и графиков.
Навыки и функциональность:

Использование dcc.Dropdown с множественным выбором (multi=True).
Применение dcc.Slider для выбора числового параметра.
Создание тепловой карты с помощью plotly.graph_objects или plotly.express.
Взаимодействие нескольких графиков и элементов управления.
Обработка и агрегация данных pandas для построения различных визуализаций.
Организация удобного и адаптивного интерфейса.
Таблицу с данными нужно сгенерировать самостоятельно любым способом в формате CSV с тем, чтобы далее загрузить их с помощью pandas и использовать в работе. Сгенерированные данные должны быть правдоподобны, корректны (например, один и тот же товар в разные дни имеет одни и те же характеристики), полны (в данных не может быть пропусков, отсутствующих значений и т.п.), репрезентативны (т.е. охватывают по возможности большой спектр из всех возможных вариантов). Объем таблиц — не менее 10000 строк в каждой.
